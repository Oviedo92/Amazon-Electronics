1. Ejecutar notebooks en /notebooks:
   - analisis_metadata.ipynb
   - analisis_reviews.ipynb

2. Objetivo del notebook:
   - Analizar datos
   - Definir reglas de limpieza

3. NO HACER en notebooks:
   - No guardar archivos finales

   - No usar chunks

4. Luego ejecutar ETL:

   python src/etl_metadata.py
   python src/etl_reviews.py

5. ETL hace:
   - Limpieza real
   - Procesamiento por chunks (500k)
   - Exportación a Parquet

6. Salida final:
   data/processed/reviews/reviews_chunk_0.parquet
   data/processed/metadata/metadata_limpio.parquet

------------------------------------------------------------
📊 ESTRATEGIA DE PROCESAMIENTO
------------------------------------------------------------

🔥 IMPORTANTE:

Ambos datasets son grandes (~10GB)

✔ REVIEWS:
   - OBLIGATORIO usar chunks
   - Dataset más crítico (20M+ registros)

✔ METADATA:
   - RECOMENDADO usar chunks
   - Aunque tenga menos registros, el tamaño es alto

------------------------------------------------------------
📦 CHUNKS (MANEJO DE MEMORIA)
------------------------------------------------------------

Tamaño recomendado:

✔ 500,000 filas → óptimo (recomendado)
✔ 100,000 filas → más seguro (PC limitada)

REGLA:

- Si el PC es bueno → usar 500k
- Si se congela → bajar a 100k

METADATA:
